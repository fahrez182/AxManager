### 使用无线调试启动

使用无线调试启动适用于 Android 11 及以上版本。此启动方法无需连接到电脑。由于系统限制，每次重启后需要重新执行启动步骤。

#### 启用无线调试

1. 搜索如何为您的设备型号启用“**开发者选项**”
2. 启用“**开发者选项**”和“**USB 调试**”
3. 进入“**无线调试**”
4. 启用“**无线调试**”

#### 启动和配对（只需一次）

1. 在 AxManager 中启动
2. [启用“**无线调试**”](#enable-wireless-debugging)
3. 在“**无线调试**”中点击“**使用配对码配对设备**”
4. 在 AxManager 的通知中输入配对码，直到显示“**配对成功**”
5. 返回 AxManager 并等待连接成功（在某些设备上您需要再次点击“**启动**”）

### 通过连接电脑启动

此启动方法适用于运行 Android 10 及以下版本的未 root 设备。遗憾的是，此启动方法需要电脑。由于系统限制，每次重启后需要重新执行启动步骤。

#### 什么是 `adb`？

Android 调试桥（`adb`）是一个多功能命令行工具，让您能够与设备进行通信。`adb 命令`可促进各种设备操作，例如安装和调试应用，并提供对 Unix shell 的访问，您可以使用它在设备上运行各种命令。

更多信息请参阅 [Android Developer](https://developer.android.google.cn/tools/adb)。

#### 安装 `adb`

1. 下载 Google 提供的“**SDK Platform Tools**"并将其解压到任意文件夹

   - Windows
   - Linux(打开新窗口)
   - Mac(打开新窗口)

2. 打开文件夹，右键选择

   - Windows 10: 在此处打开 PowerShell 窗口（按住 Shift 以显示此选项）
   - Windows 7: 在此处打开命令窗口（按住 Shift 以显示此选项）
   - Mac 或 Linux: 打开终端

3. 输入 `adb`，如果成功，您会看到一长列内容，而不是提示找不到 adb。

::: tip
请不要关闭此窗口。后文提到的“**终端**”指的是此窗口（如果您关闭了窗口，请回到[步骤 2](#install-adb)）
如果您使用 PowerShell 或 Linux/Mac，所有 `adb` 都应替换为 `./adb`
:::

#### 设置 `adb`

要使用 `adb`，您首先需要在设备上启用 USB 调试，通常按照以下步骤：

1. 搜索如何为您的设备型号启用“**开发者选项**"

2. 启用“**开发者选项**"和“**USB 调试**"

3. 将设备连接到电脑，并在终端中输入 `adb devices`。

4. 此时，设备上会出现“**允许调试**"对话框，勾选“**始终允许**"并确认。

5. 在终端中再次输入 `adb devices`。如果没有问题，您将看到类似以下内容。

   ```
   List of devices attached
   XXX     device
   ```

复制命令并粘贴到终端中。如果没有问题，您将在 AxManager 应用中看到 AxManager 已成功启动。
